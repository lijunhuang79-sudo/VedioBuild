# 把 AI 视频工厂部署成公网可访问的网页

下面用两种常见方式说明：**方式一** 用 **DigitalOcean Droplet**（或任意一台 VPS）一台机器跑全部服务；**方式二** 用 **Vercel + Railway** 等分拆部署（前端/后端分离）。

---

## 方式一：单机部署（推荐，用 DigitalOcean Droplet 或任意 VPS）

适合：希望一个域名打开就能用、成本可控、自己掌控服务器。

### 第一步：买一台云服务器（Droplet / VPS）

1. 打开 [DigitalOcean](https://www.digitalocean.com)（或 阿里云 / 腾讯云 / Vultr 等）。
2. 创建 **Droplet**（DigitalOcean 里「Create」→「Droplets」）：
   - **系统**：Ubuntu 22.04 LTS
   - **配置**：至少 2GB 内存、1 vCPU（生成视频建议 4GB+）
   - **地区**：选离你用户近的（如新加坡、旧金山）
3. 创建后记下 **公网 IP**，用 SSH 登录：
   ```bash
   ssh root@你的Droplet的IP
   ```

### 第二步：在服务器上装 Docker

```bash
# 更新系统
apt update && apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker

# 安装 Docker Compose
apt install -y docker-compose-plugin
```

### 第三步：把项目代码放到服务器

**方法 A：用 Git（推荐）**

```bash
# 若项目在 GitHub/GitLab，在服务器上克隆
cd /opt
git clone https://github.com/你的用户名/VedioBuild.git
cd VedioBuild
```

**方法 B：本地上传**

在本地项目根目录执行（把 `你的服务器IP` 换成实际 IP）：

```bash
rsync -avz --exclude 'node_modules' --exclude '.venv' --exclude '__pycache__' --exclude '.next' \
  . root@你的服务器IP:/opt/VedioBuild/
```

### 第四步：配置环境变量

```bash
cd /opt/VedioBuild
cp .env.example .env   # 若没有 .env.example，就复制你本地的 .env
nano .env
```

**必须改的项：**

| 变量 | 说明 | 示例（公网部署） |
|------|------|------------------|
| `API_BASE_URL` | 后端对外地址 | `https://api.你的域名.com` 或 `http://Droplet的IP:8000` |
| `FRONTEND_URL` | 前端对外地址 | `https://你的域名.com` 或 `http://Droplet的IP:3000` |
| `JWT_SECRET_KEY` | 生产用随机密钥 | 用 `openssl rand -hex 32` 生成 |
| `CORS_ORIGINS` | 允许的前端域名 | `https://你的域名.com,http://Droplet的IP:3000` |

数据库、Redis、MinIO 用 Docker 时保持 `docker-compose.yml` 里默认即可（postgres/redis/minio 用容器内地址）。

### 第五步：改前端请求后端的地址

若用户通过**域名**访问前端，前端请求后端要用**同一域名或 API 子域名**，不能写死 localhost。

编辑 `frontend/.env.production`（没有就新建）：

```bash
# 用域名时（推荐）
NEXT_PUBLIC_API_URL=https://api.你的域名.com

# 或暂时用 IP
NEXT_PUBLIC_API_URL=http://你的Droplet的IP:8000
```

并确认 `docker-compose.yml` 里 frontend 的 `environment` 有：

```yaml
environment:
  NEXT_PUBLIC_API_URL: http://后端地址:8000
```

把 `后端地址` 换成实际 IP 或 `api.你的域名.com`。

### 第六步：用 Docker 启动全部服务

```bash
cd /opt/VedioBuild
docker compose up -d
```

检查：

```bash
docker compose ps
```

应看到 postgres、redis、minio、backend、worker、frontend 都在运行。

### 第七步：访问网页

- **用 IP**：浏览器打开 `http://你的Droplet的IP:3000`
- **用域名**：需先做**第八步** 绑定域名并（可选）配 Nginx 反代 + HTTPS

### 第八步（可选）：绑定域名 + Nginx + HTTPS

1. 在域名服务商处把域名 **A 记录** 指到 Droplet 的 IP。
2. 在服务器上安装 Nginx 和 certbot：
   ```bash
   apt install -y nginx certbot python3-certbot-nginx
   ```
3. 配置 Nginx 反向代理（示例，把 `你的域名.com` 换成真实域名）：
   ```bash
   nano /etc/nginx/sites-available/vediobuild
   ```
   写入：

   ```nginx
   server {
       listen 80;
       server_name 你的域名.com api.你的域名.com;

       location / {
           proxy_pass http://127.0.0.1:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_cache_bypass $http_upgrade;
       }

       location /api/ {
           proxy_pass http://127.0.0.1:8000;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }

       location /static/ {
           proxy_pass http://127.0.0.1:8000;
           proxy_set_header Host $host;
       }
   }
   ```

   若前后端同域（只用一个域名），可按上面合并；若 API 用 `api.你的域名.com`，再为 API 单独写一个 `server { server_name api.你的域名.com; ... }` 把 `proxy_pass` 指到 8000。

4. 启用配置并申请 HTTPS：
   ```bash
   ln -s /etc/nginx/sites-available/vediobuild /etc/nginx/sites-enabled/
   nginx -t && systemctl reload nginx
   certbot --nginx -d 你的域名.com -d api.你的域名.com
   ```

5. 把 `.env` 里 `API_BASE_URL`、`FRONTEND_URL`、`CORS_ORIGINS` 改成 `https://...`，重启后端/前端容器：
   ```bash
   cd /opt/VedioBuild && docker compose up -d --force-recreate backend frontend
   ```

## 方式二：分拆部署（Vercel 前端 + Railway 后端）

适合：不想管服务器、希望前端自动 CI/CD、后端用托管服务。

### 1. 前端部署到 Vercel

1. 打开 [Vercel](https://vercel.com)，用 GitHub 登录。
2. 把项目推到 **GitHub**，在 Vercel 里 **Import** 该仓库。
3. **Root Directory** 选 `frontend`。
4. **Environment Variables** 增加：
   - `NEXT_PUBLIC_API_URL` = 你后端的公网地址（下一步 Railway 会给出）。
5. 部署完成后得到前端域名，如 `xxx.vercel.app`。

### 2. 后端 + Worker + 数据库部署到 Railway

1. 打开 [Railway](https://railway.app)，用 GitHub 登录。
2. **New Project** → **Deploy from GitHub**，选同一仓库（或只含 backend/worker 的仓库）。
3. 在 Railway 里添加服务：
   - **PostgreSQL**（一键添加）
   - **Redis**（一键添加）
   - **Backend**：根目录选 `backend`，Build 命令用 `pip install -r requirements.txt`，Start 命令用 `uvicorn app.main:app --host 0.0.0.0`
   - **Worker**：根目录选 `worker`，Start 命令用 `celery -A tasks worker -l info`
4. 在 Backend / Worker 的 **Variables** 里填：
   - `DATABASE_URL`（Railway PostgreSQL 提供的）
   - `REDIS_URL`（Railway Redis 提供的）
   - `JWT_SECRET_KEY`、`DEEPSEEK_API_KEY`、`API_BASE_URL`（后端公网 URL）、`FRONTEND_URL`（Vercel 的域名）等。
5. 对象存储：Railway 无内置 S3，需用 **外部 MinIO / AWS S3**，在 Variables 里配置 `S3_*`。
6. 为 Backend 生成 **公网域名**（Settings → Generate Domain），得到如 `xxx.up.railway.app`，把它填回 Vercel 的 `NEXT_PUBLIC_API_URL` 和 `API_BASE_URL`。

### 3. 注意点

- 本地用的 **进程内 Worker**（`start-backend.sh` + SQLite）在 Vercel 上不可用（Vercel 只跑前端），所以后端和 Worker 必须单独部署（如 Railway）。
- 视频文件若存在 Railway 的磁盘，重启可能丢失，建议用 S3/MinIO 存视频。

## 用「Drop」类托管（如 Droplet）小结

你提到的 **drop 网站** 通常指 **DigitalOcean Droplet**（按量付费的 VPS）。上面 **方式一** 就是「用 Droplet 把 APP 转化成网页」的完整步骤：

1. 买 Droplet（Ubuntu）
2. 装 Docker + Docker Compose
3. 把项目拷上去（Git 或 rsync）
4. 配好 `.env` 和 `NEXT_PUBLIC_API_URL`
5. `docker compose up -d`
6. 用 IP:3000 或 绑定域名 + Nginx + HTTPS 访问

这样你的 APP 就变成**任何人通过浏览器打开一个网址就能用的网页**。若你选的是别的「drop」类服务（例如只提供静态托管），可以说一下具体平台名，我可以按该平台再写一版步骤。
